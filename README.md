# MCOS / muthur-command channel version data

This repository publishes **`stable.json`**, **`beta.json`**, and **`dev.json`**, plus signature sidecars (`.sig`) and static AppArmor profiles, for **Supervisor**, **OS**, and **mcio pre-pull** consumers.

Before enabling the release workflow, set in **`version.yml`** → **`env`**: **`MCOS_VERSION_S3_BUCKET`** (bucket name; must match the name exposed in AList S3 mapping if you use AList), **`MCOS_VERSION_S3_PUBLIC_BASE`** (passed to **`aws s3`** as **`--endpoint-url`**—your S3-compatible API base), **`MCOS_VERSION_PUBLIC_BASE`** (HTTPS base for clients fetching e.g. `https://…/stable.json` and static assets), and **repository Actions secrets** **`AWS_ACCESS_KEY_ID`** / **`AWS_SECRET_ACCESS_KEY`** (AList S3 keys or any compatible gateway). **If these secrets are empty, `aws s3 sync` fails with `Unable to locate credentials`.** Add them under **Settings → Secrets and variables → Actions → New repository secret** (forks do not inherit secrets from the upstream repo). You may point **`MCOS_VERSION_S3_PUBLIC_BASE`** and **`MCOS_VERSION_PUBLIC_BASE`** at the same host when one endpoint serves both the S3 API and public HTTP, or keep them separate (e.g. regional S3 API vs a CDN or vanity domain for downloads).

**TLS errors (`SSL validation failed` / `TLSV1_UNRECOGNIZED_NAME`):** the hostname in **`MCOS_VERSION_S3_PUBLIC_BASE`** must present a certificate whose **SAN/CN matches** that host (correct **SNI** on the reverse proxy). Fix TLS on the server or use a publicly trusted cert (e.g. Let’s Encrypt). **Temporary lab-only workaround:** add a repository **Variable** **`MCOS_S3_NO_VERIFY_SSL`** = **`true`** (Actions → Variables); the workflow will pass **`aws --no-verify-ssl`**. Remove it in production.
